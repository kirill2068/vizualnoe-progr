## Массив: непрерывность и предсказуемость

### Схематическое представление массива в памяти

```
Адрес:  0x1000    0x1004    0x1008    0x100C    0x1010
       +--------+--------+--------+--------+--------+
Данные: |   A[0] |   A[1] |   A[2] |   A[3] |   A[4] |
       +--------+--------+--------+--------+--------+
Индекс:    0        1        2        3        4
```

### Как устроен массив

Массив представляет собой **непрерывный блок памяти**, где все элементы расположены последовательно друг за другом. Каждый элемент имеет одинаковый размер, что позволяет вычислять адрес любого элемента по формуле:

```
адрес_элемента[i] = начальный_адрес + i × размер_элемента
```

**Ключевые характеристики:**
- **Непрерывность памяти** — все элементы занимают смежные ячейки
- **Фиксированный размер** — размер массива определяется при создании
- **Быстрый доступ** — обращение к любому элементу за константное время O(1)
- **Кэш-дружественность** — благодаря локализации данных хорошо работает с кэшем процессора

### Пример в коде 

```kotlin
val array = arrayOf(10, 20, 30, 40, 50)
val intArray = intArrayOf(10, 20, 30, 40, 50)

println(array[0])  // 10 - доступ за O(1)
println(array[2])  // 30

array[1] = 25

for (i in array.indices) {
    println("array[$i] = ${array[i]}")
}

val matrix = Array(3) { IntArray(3) }
```

## Связный список: гибкость и динамичность

### Схематическое представление связного списка в памяти

```
Голова списка
    │
    ▼
+-------------+-------------+    +-------------+-------------+    +-------------+-------------+
| Данные: 10  | Next: 0x3000| -->| Данные: 20  | Next: 0x4000| -->| Данные: 30  | Next: NULL  |
+-------------+-------------+    +-------------+-------------+    +-------------+-------------+
Адрес: 0x2000                    Адрес: 0x3000                    Адрес: 0x4000
```

### Как устроен связный список

Связный список состоит из узлов, разбросанных в произвольных местах памяти. Каждый узел содержит:

1. **Полезные данные** — хранимая информация
2. **Указатель/ссылка** — адрес следующего узла в списке

**Структура узла:**
```kotlin
class Node<T>(
    var data: T,
    var next: Node<T>? = null
) {
    override fun toString(): String {
        return "Node(data=$data, next=${next?.data})"
    }
}
```

**Ключевые характеристики:**
- **Динамическое расширение** — размер может меняться во время выполнения
- **Непрерывность не требуется** — узлы могут быть разбросаны по памяти
- **Гибкость вставки/удаления** — изменение только указателей
- **Последовательный доступ** — для доступа к элементу нужно пройти все предыдущие

### Пример в коде 

```kotlin
// Создание узлов и формирование списка
val node1 = Node(10)
val node2 = Node(20)
val node3 = Node(30)

node1.next = node2
node2.next = node3

// Альтернативный способ создания
val head = Node(10, Node(20, Node(30)))

// Функция для печати списка
fun <T> printList(head: Node<T>?) {
    var current = head
    while (current != null) {
        print("${current.data} -> ")
        current = current.next
    }
    println("null")
}

// Использование стандартного LinkedList из Kotlin
val linkedList = LinkedList<Int>()
linkedList.add(10)
linkedList.add(20)
linkedList.add(30)

// Вставка в начало
linkedList.addFirst(5)

// Удаление из конца
linkedList.removeLast()
```

## Сравнительный анализ

### Таблица сравнения

| Критерий | Массив | Связный список |
|----------|--------|----------------|
| **Организация памяти** | Непрерывный блок | Фрагментированные узлы |
| **Размер** | Фиксированный | Динамический |
| **Доступ к элементу** | O(1) — по индексу | O(n) — последовательный |
| **Вставка в начало** | O(n) — сдвиг элементов | O(1) — изменение указателя |
| **Удаление из начала** | O(n) — сдвиг элементов | O(1) — изменение указателя |
| **Вставка в конец** | O(1) — если есть место | O(n) — нужно найти конец |
| **Использование памяти** | Только под данные | Данные + указатели |
| **Кэш-эффективность** | Высокая | Низкая |

### Визуальное сравнение операций

**Вставка в массив:**
```
До:    [A][B][C][D][ ][ ]
Вставка X на позицию 1:
После: [A][X][B][C][D][ ]
       (все элементы сдвигаются)
```

**Вставка в список:**
```
До:    A → B → C → D
Вставка X после A:
После: A → X → B → C → D
       (меняются только указатели)
```
